# Code of Conduct

## Our standard

We want Downmark to be a calm, respectful project where contributors can ask questions, share ideas, and disagree constructively.

Examples of behavior that help:

- being kind and patient
- assuming good intent
- giving clear, actionable feedback
- respecting different levels of experience

Examples of behavior that are not acceptable:

- harassment or personal attacks
- discriminatory language
- doxxing or sharing private information
- trolling, intimidation, or bad-faith disruption

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or pull requests that violate this code of conduct.

If you experience or witness unacceptable behavior, report it privately to the maintainers through GitHub security/contact channels rather than escalating publicly.
